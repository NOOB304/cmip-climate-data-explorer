# CMIP Climate Explorer 中文用户指南

## 1. 选择存储位置

首次运行时，如电脑有 D 盘，气候文件默认保存在 `D:\CMIP Climate Explorer`。
在“设置”页可以改为其他磁盘或目录，并可选择是否在下载后自动转换 GeoTIFF。
程序配置和日志仍位于 Windows 应用数据目录，但不会把大型气候文件放在那里。

## 2. 查询与下载

1. 在“数据下载”页输入变量名称或代码进行筛选，再从“变量”下拉框选择变量。
2. 模型、情景、数据表、频率和网格都从下拉框选择；“全部”表示不限制该条件。
3. 输入需要的开始和结束年份。只要文件覆盖范围与所选年份有交集就会显示。
4. 点击“查询数据”，勾选一个或多个文件。选中行会高亮，顶部显示已选数量。
5. 点击“下载所选”。下载继续在后台运行，可转到“下载任务”查看状态。

程序优先使用 HTTPS，并自动切换镜像。对于证书过期的旧节点，仅在有 ESGF 校验和
可验证文件完整性时自动回退到 HTTP。中断后可在任务页恢复下载。

## 3. 查看任务

“下载任务”页每秒刷新。顶部提示当前运行任务数，每行先显示状态和实际字节进度，再显示
省略后的文件名和保存目录。选中任务可在下方查看完整信息，右键可在资源管理器中定位。
可暂停、恢复、取消，失败或意外中断后可点击“重新连接”。网络短暂断开时程序会先在同一
任务中自动重连并继续 `.part` 断点，不会新增任务记录。“取消”只影响选中任务，
“全部停止”会同时停止当前下载和全部等待任务。

NetCDF 默认保存到 `NetCDF\模型\情景`，自动生成的 GeoTIFF 保存到
`GeoTIFF\模型\情景`。

月、年和固定场数据可以自动转 TIF。3 小时、6 小时和日数据会保留为 NC，避免一次生成
成千上万个 TIF；需要时可在“文件处理”中聚合后输出。

## 4. 文件处理

1. 在“文件处理”页选择一个或多个网格兼容的 `.nc` 或 `.nc4` 文件。
2. 程序自动读取可处理变量、时间频率和覆盖年份；无需手工填写变量代码或年份。
3. 月数据会显示 1-12 月复选框。选择参与合成的月份和年度统计方式：平均、总和、
   最大、最小或众数。
4. 选择输出位置并点击“开始处理”。下方进度条按生成的年度文件更新。

输出为 EPSG:4326 GeoTIFF/COG，缺失值写为 NoData。

## 5. 本地数据与日志

“本地数据”页扫描设置中的存储目录。双击文件或右键选择“在资源管理器中显示”，也可
校验 SHA-256 或删除文件。

“操作日志”记录查询、下载、任务控制、文件处理、打开目录、校验、删除和设置变更。

## 6. 常见问题

- **任务长时间显示正在连接**：程序会依次尝试镜像；任务页保持运行提示。若全部失败，
  任务会明确显示失败，可稍后重试。
- **进度条是不定进度**：数据节点未提供文件大小时会先显示活动动画；收到大小后切换为
  百分比进度。
- **自动转换失败但 NC 已下载**：在“本地数据”确认 NC 文件存在，再到“文件处理”手动
  选择变量进行转换。
- **文件没有中文名称**：优先显示内置中文名称；未收录时显示 CMIP 官方英文长名称，
  不再显示空白或横线。
